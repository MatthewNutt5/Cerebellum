QoL Features ==========

- Procedural GUI
    - On startup, the GUI reads each class implementing the Device interface
        (ignoring PowerSupply) and offers a dropdown menu of options for Configs
    - The config field types are already provided by the corresponding Config,
        and it can additionally specify a selection of options - the GUI uses
        this for its display
    - Fields can also specify a name to display, e.g. ip --> "IP Address"
    - In both of these cases, the spec will exist in a predetermined name as a
        class attribute, e.g. self.field --> field_options and field_title
        - If the name is not found, default to an open box and the field name,
            respectively
    - Special cases
        - Any field named "com" will use pyserial to grab all available COM ports
- Persistence of GUI state
