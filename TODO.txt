Feature Ideas ==========

Core Features ------

- Pre-compiled installation
    - Compile core Cerebellum into an .exe, but still retain the Device
        submodule and Event package as dynamically imported libraries, to
        preserve customizability
    - May be a good idea to break Event.py into an Event submodule, to help with
        organization and make it easier to update events
- Add storage of test results (e.g. csv file)
    - Events no longer return anything to run_test, so maybe start a csv in
        run_test, pass it to event.exec(), and allow them to write outputs?
- Test preferences
    - Verbosity level, runtime error behavior, save test results to file
- Add automatic script for setting up Tamalero enviroment variables (i.e. remove
    the need to manually run setup.sh every time)
- Pause on exception in run_test instead of immediate test abort
    - Could have an (Ignore) / Pause / Abort option in preferences
- Add log-to-file
    - Captures entire log output, instead of selective csv writes

GUI Features -----

- Add device indices to EnvironmentConfig (e.g. instead of "DeviceConfig", it 
    says "Device #0", etc.)
- Allow for moving devices and events up/down
- Add event device_idx remapper to EventConfigGUI
    - What if someone reorders their devices and breaks the device_idx-es?
        What if they get a TestConfig from someone else with a different setup?
- Persistence of GUI state
- Test progress bar
- Fix QComboBox flickering?

Devices -----

- Add MCU (microcontroller) device
    - Allows for reading and writing to/from pins
    - Could be used to trigger interlocks, read voltages, etc.
    - Would have implementations like PowerSupply
- Add Oscilloscope device
    - Allows for waveform retrieval
    - Would have implementations like PowerSupply
- Add Wiener crate support through the AnaGate (as a PowerSupply)
    - Use AnaGate dll to send/receive CAN commands
