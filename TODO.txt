Fixes/Refactors ==========

- Add more comments and update existing ones
- Write a README with details on:
    - Standard GUI operation
    - Custom implementations of devices, power supplies, and events
- Rewrite RunTestGUI with QProcess and a text box
    - Current setup doesn't allow for concurrent GUI control over a running
        test, nor input from the terminal (at bare minimum, for WaitEvents)
    - Use a QProcess that connects to the log_box, and a LineEdit below(?) that
        will pipe to the process
- Add more checks to read_json methods
    - We check whether or not a good constructor can be found, but what if the
        JSON doesn't even include a name to look with?
- Consider reorganizing Event into a subpackage to make adding new Events easier
    - For instance, start with a base Event module that includes the abstract
        classes and device-less events, then have another module for PowerSupply
        Events, one for RB events, etc.
    - This way, users can simply add new modules as needed, instead of having
        to update the monolithic Event.py file

    - At first, the idea was to use external scripts, but Cerebellum is built

    - Could be used to trigger interlocks, read voltages, etc.
    - Would have implementations like PowerSupply
- Add Oscilloscope device
    - Allows for waveform retrieval
    - Would have implementations like PowerSupply
- Add Wiener crate support through the AnaGate (as a PowerSupply)
    - Use AnaGate dll to send/receive CAN commands
